# Topic Output Format

本文件在 script-writer 的**入口 B（topic 研究模式）**的阶段一使用。

## 阶段一结构

严格按这个顺序输出：

1. Input understanding
2. Research base
3. First-layer candidate pool
4. Second-layer expanded picks
5. Stage-one checkpoint

## 1. Input understanding

先说明：

- whether the input is a topic or an existing draft
- the original viewpoint if a draft was provided
- the extension opportunity
- the opposing angle

如果用户给的是现有文稿，这一段不能只复述原文，必须点出"还能往哪延展"和"最成立的对立面是什么"。

## 2. Research base

简洁展示 `research-framework.md` 的核心结果，用于支持选题判断，不要在这里再次长篇重讲研究方法。

- identify the core factual anchors that shape the topic
- note the most relevant audience context
- surface any obvious constraints or missing information
- keep this section short and usable for comparison

## 3. First-layer candidate pool

Generate 5-8 short candidate directions.

每个方向都要包含：

- topic name
- angle type: original / extension / opposing
- why it is worth covering
- likely audience pull
- risk or failure mode

这些方向必须拉开差异，避免"只是同一个观点换标题"。

## 4. Second-layer expanded picks

Automatically choose the 3 strongest options and expand each one with:

- working title
- one-sentence thesis
- why it has distribution potential
- short research summary
- possible angle path
- backup title ideas
- possible controversy or tension points
- fit for audience/context
- suggested expression temperament

优先选"值得继续做"的方向，而不是只选"最安全"的方向。

在推荐中可初步判断最适合的 harness Platform 规格（9:16 / 16:9 / 4:5）和 Mood（Impressed / Excited / Calm / Inspired），供阶段二确认。

## 5. Stage-one checkpoint

End with a clear request for the user to:

- choose one of the 3 expanded directions
- or request a modification to one of them

这一段必须停住，不要在阶段一直接生成最终口播稿。
